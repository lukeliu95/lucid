## 1. 项目背景

Alan 是 AI / 商业重度信息消费者,每天面对 1-3 小时长访谈/播客/纪录片"看不完"。市面 AI 摘要工具多为单条 Chrome 插件式,缺少**结构化入口 + 双语 + 时间轴 + 可分享详情页**。kdsj-world 把"长内容解释器"做成可分享的 Web 入口,MVP 锁定 AI/科技/商业访谈领域,30 条 demo 验证假设。

---

## 2. 目标与范围

### 2.1 目标

- G1 Alan 自用(Cooper 第一用户)
- G2 一条视频 → 两分钟读懂(摘要 + 5-10 观点 + 时间轴 + 2 档解释)
- G3 中英双语 MVP(UI + 内容摘要双版)
- G4 5-31 上线 + 30 demo 视频 + Twitter 传播
- G5 长期承载 Simprr 品牌(footer 链接)

### 2.2 范围

**包含(MVP)**:
- 4 个主页面(首页 / 视频详情 / 人物页 / 主题页)+ 搜索 + 用户收藏
- 30 视频 + 10 人物 + 5 主题数据
- DeepSeek-v4-pro 摘要/观点/时间轴/2 档解释流水线 + 本地 Whisper ASR
- 中英双语切换(next-intl)
- 极简后台(视频 CRUD + AI 状态查看)

**不包含(MVP 砍)**:
- 用户提交链接 + 队列(v1.1+)
- 复杂推荐算法(编辑精选 + 时间倒序替代)
- 移动端响应式(桌面 web 优先)
- 完整后台 8 模块(只保留 CRUD + 状态)
- 知识图谱 / 学习路径(v2.0)
- 多领域(MVP 只 AI/科技/商业)
- 5 档解释(只 2 档)
- 1000 视频量(只 30)

## 4. 功能需求(6 模块 · 21 REQ)

### 4.1 模块 H · 首页(Home)

#### REQ-H01 · 精选 30 视频展示
- **优先级**: P0
- **用户故事**: 作为访客,我希望首页一目了然看到精选视频卡片,知道这站讲什么。
- **输入**: 无(URL `/`)
- **输出**: 视频卡片 grid(封面 + 标题 + 嘉宾 + 时长 + 主题 tag + 一句话摘要)
- **交互**: 点击 → 详情页 · 主题 tag → 主题页
- **对应 Goal**: G2/G4
- **实现类型**: UI(渲染) + Script(数据查询)
- **验收**: 桌面 1440 宽度 grid 显示 ≥ 12 卡片首屏 · 时间倒序 · footer 含 simprr 链接

#### REQ-H02 · 双语切换 toggle
- **优先级**: P0
- **用户故事**: 作为中英用户,我希望右上角 toggle 一键切换全站语言。
- **输入**: 用户点击 toggle
- **输出**: URL 切到 `/en` 或 `/zh` · 全站 UI + 内容文字立刻变
- **交互**: next-intl 路由
- **对应 Goal**: G3
- **实现类型**: UI
- **验收**: 100% i18n key coverage(eval) · 无未翻译 fallback

#### REQ-H03 · 主题导航 + 热门人物
- **优先级**: P1
- **用户故事**: 作为访客,我希望在首页能看到 5 主题 + 部分人物入口。
- **输入**: 无
- **输出**: 主题胶囊条(5 个)+ 人物 avatar 列(6-10 个)
- **实现类型**: UI + Script
- **验收**: 点击主题/人物 → 对应聚合页

---

### 4.2 模块 V · 视频详情页(Video Detail)— 核心

#### REQ-V01 · 视频嵌入播放器
- **优先级**: P0
- **用户故事**: 作为访客,我希望页面顶部能直接播放原视频(YouTube/B 站嵌入)。
- **输入**: video.platform_id + platform
- **输出**: iframe embed
- **实现类型**: UI
- **验收**: 30 视频每条 embed 都能播 · 无版权报错

#### REQ-V02 · 一句话总结
- **优先级**: P0
- **用户故事**: 作为信息消费者,我希望进页 3 秒内读到"这条讲了啥"。
- **输入**: ASR 转录(英文)
- **输出**: ≤ 80 字双语一句话总结
- **实现类型**: Agent(DeepSeek-v4-pro)
- **验收**: 准确率 ≥ 85%(20 case 抽检 · Phase C)

#### REQ-V03 · 核心观点 5-10 条
- **优先级**: P0
- **用户故事**: 作为读者,我希望扫一眼就抓住这条视频的 5-10 个观点。
- **输入**: ASR 转录
- **输出**: 5-10 条 bullet · 双语
- **实现类型**: Agent
- **验收**: 幻觉率 ≤ 5%(无来源虚构) · 每条 ≤ 50 字

#### REQ-V04 · 时间轴(≥5 节点)
- **优先级**: P0
- **用户故事**: 作为想精读的用户,我希望按时间轴跳到关键节点深听。
- **输入**: ASR 带时间戳的分段
- **输出**: ≥ 5 个节点(timestamp + 标题 + 一句摘要)· 双语
- **交互**: 点击 timestamp → 跳转视频对应秒
- **实现类型**: Hybrid(Script 分段 + Agent 命名)
- **验收**: 可用率 ≥ 80% · 分段合理性人工评

#### REQ-V05 · 2 档解释(成人速懂 / 深度版)
- **优先级**: P0
- **用户故事**: 作为不同消化深度的读者,我希望选"快版"或"深度版"。
- **输入**: ASR 转录 + 视频元数据
- **输出**: 双语 · 成人速懂 (300 字内) + 深度版(800-1500 字)
- **交互**: tab 切换
- **实现类型**: Agent
- **验收**: 两档语气 / 信息密度有可见差异

#### REQ-V06 · 视频内双语切换
- **优先级**: P0
- **用户故事**: 详情页所有 AI 产出内容(总结/观点/时间轴/解释)双语 toggle 无缺漏。
- **输入**: locale
- **输出**: 对应语言版本
- **实现类型**: UI + Script(读 DB 中预生成的双语字段)
- **验收**: 双语 100% 覆盖 · 0 fallback

#### REQ-V07 · 来源信息 + 版权链
- **优先级**: P0
- **用户故事**: 作为内容方,我希望页面清楚标原作者/频道/平台/原链接。
- **输入**: video.metadata
- **输出**: 元信息卡片 + "在 YouTube 看原片"外链
- **实现类型**: UI
- **验收**: 30 条全有 · 0 缺失

---

### 4.3 模块 P · 人物页(Person)

#### REQ-P01 · 人物详情(10 位)
- **优先级**: P0
- **用户故事**: 作为追某位 KOL 的访客,我希望看到他的简介 + 代表观点 + 相关视频。
- **输入**: person_id
- **输出**: 头像 + 双语简介 + 代表观点 3-5 条 + 相关视频卡片
- **实现类型**: UI + Script(JOIN videos by person)
- **验收**: 10 位人物全有数据 · 每位至少 1 条相关视频

#### REQ-P02 · 人物代表观点生成
- **优先级**: P1
- **用户故事**: 作为运营,我希望系统能基于该人物所有视频自动抽他的代表观点。
- **输入**: 该人物所有视频的核心观点 JOIN
- **输出**: 3-5 条代表观点 · 双语
- **实现类型**: Agent(基于 REQ-V03 聚合)
- **验收**: 准确率 ≥ 85% · 不重复

---

### 4.4 模块 T · 主题页(Topic)

#### REQ-T01 · 主题聚合(5 个)
- **优先级**: P0
- **用户故事**: 作为关注 "AI Agent" 的访客,我希望看到该主题下所有视频。
- **输入**: topic_slug(`ai` / `ai-agent` / `startup` / `chip` / `future-of-work`)
- **输出**: 双语主题简介 + 该主题下视频卡片
- **实现类型**: UI + Script
- **验收**: 5 主题各 ≥ 3 视频

---

### 4.5 模块 S · 搜索(Search)

#### REQ-S01 · 关键词搜索
- **优先级**: P0
- **用户故事**: 作为想找特定主题/人物的用户,我希望关键词搜索能命中标题/人物/主题。
- **输入**: query string
- **输出**: 命中视频/人物/主题列表
- **实现类型**: Script(Postgres ILIKE / ts_vector)
- **验收**: 返回 ≤ 500ms · 双语字段都索引

#### REQ-S02 · 简易语义搜索
- **优先级**: P1
- **用户故事**: 作为输入"长上下文"的用户,我希望能命中讨论该概念的视频。
- **输入**: query
- **输出**: pgvector top-k 视频
- **实现类型**: Hybrid(Script embed query + vector search · embedding 用 DeepSeek 或 OSS)
- **验收**: top-5 含有 ≥ 1 相关视频(主观评)

---

### 4.6 模块 U · 用户与收藏(User)

#### REQ-U01 · 登录(简单)
- **优先级**: P0
- **用户故事**: 作为想收藏的用户,我希望能用邮件/Google 登录。
- **输入**: 邮箱或 OAuth
- **输出**: session
- **实现类型**: UI + Script(NextAuth / Clerk · 老周裁)
- **验收**: 登录成功率 ≥ 95%

#### REQ-U02 · 收藏视频
- **优先级**: P0
- **用户故事**: 作为登录用户,我希望点心标 → 加入我的收藏。
- **输入**: user_id + video_id
- **输出**: favorite 记录
- **实现类型**: Script(CRUD)
- **验收**: 点击即时反馈

#### REQ-U03 · 个人主页
- **优先级**: P0
- **用户故事**: 作为登录用户,我希望有 `/me` 看自己收藏列表。
- **输入**: session
- **输出**: 收藏视频 grid
- **实现类型**: UI + Script
- **验收**: 收藏可分页

---

### 4.7 模块 X · 极简后台(Admin)

#### REQ-X01 · 视频 CRUD
- **优先级**: P0
- **用户故事**: 作为运营(Alan),我希望后台能加/改/删视频条目。
- **输入**: 视频元数据 + 嘉宾 + 主题
- **输出**: DB 写入 · 触发 AI 流水线
- **实现类型**: UI + Script
- **验收**: 加一条视频 < 1 分钟手工录入 · 自动触发 AI 处理

#### REQ-X02 · AI 处理状态查看
- **优先级**: P0
- **用户故事**: 作为运营,我希望看到每条视频的 ASR / 摘要 / 翻译状态。
- **输入**: 无
- **输出**: 视频列表 + status 列(pending / asr_done / ai_done / failed)
- **实现类型**: UI + Script
- **验收**: 状态准实时(< 5s 滞后)

#### REQ-X03 · 摘要校对(轻量)
- **优先级**: P1
- **用户故事**: 作为运营,我希望能编辑 AI 产出的摘要文本修错。
- **输入**: video_id + 字段
- **输出**: DB 更新
- **实现类型**: UI + Script
- **验收**: 保存后前台立即更新
