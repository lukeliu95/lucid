## 3. Stage C · 视觉策略

### 3.1 Brand Alignment

**主参考**: **Electronic Magazine × E-Ink mixed**(`design-library/voltagent` 风格的"科技理性"做辅助)

理由:
- "看懂世界"语义 → 杂志阅读 → 衬线字体承载阅读权威感
- E-Ink 暖白(#F8F5EE)+ 墨黑(#1A1612)+ 暖橘高亮(#D97706)→ 信息密度高且不刺眼
- 衬线长文阅读疲劳低,适合深度版 800-1500 字
- 避免 SaaS-Generic 的渐变紫蓝,与"读物"调性吻合

### 3.2 Tokens 决策(完整在 `tokens/`)

- 字体: 中文 `Source Han Serif SC` / 英文 `Source Serif Pro` / mono `JR Mono`(代码不多)
- 主色:墨黑 #1A1612(text) / 暖白 #F8F5EE(bg) / 暖橘 #D97706(accent · 链接 + CTA + 高亮节点)
- 状态色:成功 #15803D / 警告 #CA8A04 / 错误 #B91C1C / info #1E40AF
- Radius:全站 4px(锐 · 杂志感)· 卡片图片角 8px
- Shadow:几乎无 · 用细描边 `1px solid #E5DFD0` 替代(E-Ink 平面感)

## 6. Stage D · 关键交互(完整 spec 见 interactions.md)

6 大关键交互:
1. **2 档解释 tab 切换** — 无 layout shift · 内容渐入 150ms
2. **时间轴跳秒** — 点击节点 → iframe postMessage seekTo(seconds)+ 节点高亮 250ms
3. **双语切换** — next-intl 路由切换 + 内容字段切换 · 无 FOUC
4. **搜索** — 输入 300ms debounce · ESC 关闭 · `/` 快捷聚焦
5. **收藏** — 未登录态点 → 跳 `/login?redirect=...` · 登录态 → 心标填充 + 微反馈
6. **运营状态轮询** — `/admin` 列表 30s 轮询 · status 色块更新

## 7. 视觉硬门槛(自检 + 大刘验收)

- [ ] 字体: 衬线主用 · sans 仅在 UI label / button
- [ ] 暖橘 #D97706 是唯一 accent · 不出现紫蓝渐变
- [ ] 卡片用细描边 1px solid #E5DFD0 · 不用 shadow
- [ ] 长文 max-width 680px(深度版正文)
- [ ] line-height: 标题 1.25 · 正文 1.65
- [ ] 详情页主内容 col 8 (~ 736px) · 侧栏 col 4 (~ 352px) gap 32
- [ ] 全站 grid: 12 col · gutter 24 · margin 64
- [ ] LocaleToggle 切换不引起 layout shift(中英字符宽度差 → 给固定宽容器)
